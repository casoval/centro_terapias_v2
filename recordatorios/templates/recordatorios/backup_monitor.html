Respaldo automático diario

Cron · 03:00 AM

Todos los días a las 3:00 AM (hora del servidor) se ejecuta automáticamente un respaldo completo de la base de datos PostgreSQL. El proceso genera un archivo .dump comprimido y lo envía por correo.

0 3 * * * /var/www/centro_terapias/venv/bin/python /var/www/centro_terapias/backup_db.py
📧

Envío por correo

El archivo de respaldo se adjunta y envía a los correos configurados vía Gmail SMTP. El correo incluye la fecha, tamaño del archivo y las instrucciones para restaurar la base de datos.

📨 centroinfantilmisaelbolivia@gmail.com
📨 jorgeosvaldoch111@gmail.com

⚠️ Gmail rechaza adjuntos mayores a 25 MB. El archivo actual pesa menos de 1 MB.

🗄️

Formato del archivo

Se usa el formato PostgreSQL Custom (.dump) con compresión zlib nivel 6. Este formato permite restaurar tablas específicas sin necesidad de restaurar toda la base de datos.

# Restaurar base de datos completa:
pg_restore -h HOST -U centro_user -d centro_misael backup_centro_misael_FECHA.dump

# Restaurar solo una tabla:
pg_restore -h HOST -U centro_user -d centro_misael -t nombre_tabla backup.dump
💾

Archivos locales
